## 題目 4：遞迴的足跡 (Footsteps of Recursion) - 30分

### 故事背景

費波那契數列的遞迴定義雖然優雅，但電腦實際是怎麼計算的呢？為了讓大家直觀地看到「遞迴樹」的展開與回溯過程，這次我們來做一個漂亮的視覺化輸出，看看程式是如何一步步「爬樹」找答案的！

### 題目說明

讀取一個整數 `n`，計算第 `n` 項費波那契數 $F(n)$。

定義（從 1 開始）：

- $F(1) = 1$
- $F(2) = 1$
- $F(n) = F(n-1) + F(n-2)$

**要求：**

1. **必須**實作並使用遞迴函數：
    - `int fib(...)`：計算費波那契數並處理縮排輸出（請自行設計需要的參數）。
2. **限制：** 本題**嚴格禁止使用任何迴圈** (for, while, do-while)，違者不予計分。助教將會檢查關鍵字。
3. **進入函數時**：先印出縮排，再印出 `SEARCH fib(n)`。
4. **離開函數前**：先印出縮排，再印出 `GET fib(n) = [結果]`。
5. **縮排規則**：根據目前的遞迴深度印出對應數量的 `|--`。例如深度 1 印出 1 個 `|--`，深度 2 印出 2 個 `|--`，以此類推。（深度 0 為最外層，不需印出 `|--`）。
6. 務必**先呼叫計算 F(n-1)，再呼叫計算 F(n-2)**。

### 輸入格式

一行包含一個整數：`n` (1 ≤ n ≤ 10)

### 輸出格式

- 過程：依照範例印出遞迴呼叫與回傳的過程
- 最後一行：印出最終計算結果

### 範例

**Sample Input 1**

```jsx
3
```

**Sample Output 1**

```jsx
SEARCH fib(3)
|--SEARCH fib(2)
|--GET fib(2) = 1
|--SEARCH fib(1)
|--GET fib(1) = 1
GET fib(3) = 2
2
```

**Sample Input 2**

```jsx
4
```

**Sample Output 2**

```jsx
SEARCH fib(4)
|--SEARCH fib(3)
|--|--SEARCH fib(2)
|--|--GET fib(2) = 1
|--|--SEARCH fib(1)
|--|--GET fib(1) = 1
|--GET fib(3) = 2
|--SEARCH fib(2)
|--GET fib(2) = 1
GET fib(4) = 3
3
```
