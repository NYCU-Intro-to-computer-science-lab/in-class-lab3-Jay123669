## 題目 3：破解數學家的謎題 (Cracking the Mathematician's Puzzle) - 30分

### 故事背景

著名的哥德巴赫猜想 (Goldbach's Conjecture) 提出：「任一大於 2 的偶數，都可以表示成兩個質數之和」。例如 $4 = 2+2$，$6 = 3+3$，$8 = 3+5$。為了驗證這個猜想，我們需要一個程式來找出這兩個質數！

### 題目說明

讀取一個偶數 `n`，找出兩個質數 `a` 和 `b`，使得 `a + b = n`。

為了找到這組解，你需要找過所有可能的組合，並檢查 `i` 和 `n-i` 是否**同時**為質數。

**要求：**

1. 為了避免在迴圈中重複撰寫兩次質數判斷的邏輯，**必須**實作並使用一個純函數：
    - `bool isPrime(int n)`：回傳 `n` 是否為質數。
2. **限制：** 請勿使用陣列 (Array) 或 `vector` 製作質數篩表 (Sieve of Eratosthenes)，請直接在 `isPrime` 函數中透過迴圈進行判斷。

### 輸入格式

一行包含一個偶數：`n` (4 ≤ n ≤ 1,000,000)

### 輸出格式

一行：兩個質數 `a` 和 `b`，以空格分隔，使得 `a + b = n` 且 `a ≤ b`。

若有多組解，請輸出 `a` 最小的那一組。

### 範例

**Sample Input 1**

```jsx
10
```

**Sample Output 1**

```jsx
3 7
```

**Sample Input 2**

```jsx
100
```

**Sample Output 2**

```jsx
3 97
```

### 提示

- 只要檢查到 `n/2` 即可。
